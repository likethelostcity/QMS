Work phase

Where in the lifecycle
Maps to the four-gate D&D framework. Tells you what stage of product development the artifact belongs to.

Artifact type

What kind of working document
Describes what the file actually is in the working context — deliberately different from QMS document types.

Work status

Where the artifact stands
Tracks the artifact's own lifecycle, not document control state. "Promoted to QMS" closes the loop back to the controlled register.
